# 预测市场应用规格

## 基于 OCP（承诺证明 POC）的预测市场

> 本文档描述在**共识上链协议（OCP）**·**承诺证明（Proof of Commitment, POC）**之上构建的预测市场应用。OCP 的共识权威性由**资本的可验证锁定**证明，不依赖预言机或身份。协议设计见 [OCP 白皮书](./OCP_WHITEPAPER_CN.md)、[CCP 白皮书](./CAPITAL_COMMITMENT_PROTOCOL_WHITEPAPER_CN.md)。本应用严格遵循 OCP 的**一次性终局**规则（冷静期/再投票期仅一轮）。

---

## 1. 应用概述

预测市场是「**质押—挑战—共识**」三阶段流程的完整实例化。在 CCP 之上增加：

- **金库**：存款即质押权重，锁定至结算
- **预测市场**：交易 YES/NO 份额，手续费注入金库
- **分红**：手续费按本金占比分配；可选**躺平者惩罚**以激励流动性（见 2.3，非 OCP 协议要求）
- **结算**：按 OCP 规则，冷静期/再投票期一次性终局后形成最终共识

### 1.1 流程（质押—挑战—共识）

```
事件创建 → 开放质押期（至 t₀）：预测市场交易 + 金库存款（锁定），形成初始信念分布
        ↓
t₀ 截止 → 冷静期/挑战窗口（Δ）：Y/N 劣势方可支付保证金 b 发起挑战；任何人可发起 INVALID 挑战
        ↓
无人挑战 → 协议终局，输出最终共识（Finalized Consensus），领先方赢得全部
有人挑战 → 开放再投票期（Δ）：PM + 金库开放，可流入 YES/NO/INVALID，参与者可重新投票；不再接受任何新挑战
        ↓
再投票期结束 → 协议终局：YES>50% 或 NO>50% 则该方通吃，否则 INVALID 全员退款
```

## 2. 核心机制

### 2.1 基础结构

- **三态市场**：事件结果 YES、NO 或 INVALID（遵循 OCP 协议层）
- **金库**：存款即投票权重，锁定至最终结算
- **预测市场**：交易 YES/NO 份额，手续费注入金库
- **分红**：手续费按本金占比分配；可选躺平者惩罚（见 2.3）

### 2.2 关键规则

| 规则 | 说明 |
|------|------|
| 锁定 | 金库存款在最终结算前不可提取（POC：共识权威性由资本可验证锁定证明） |
| Y/N 挑战 | 仅劣势方（YES/NO 中资金少的一方）可发起；INVALID 挑战任何人可发起（与 OCP §2.2 一致） |
| 一次性终局 | 冷静期内仅可发起一次挑战；有人挑战后进入再投票期，**不再接受任何新挑战**，再投票期结束后协议立即终局（与 OCP §2.5 一致） |
| 保证金 b | b = 金库最小存款额（协议定义，金库发起人设定；参与门槛与挑战门槛对称）；赢则退还，输则处置见 OCP §2.6 |
| 再投票期 | 预测市场与金库均可交易/存入；**参与者可重新投票、重新选择立场**；新资本可流入 YES/NO/INVALID |

### 2.3 躺平惩罚与分红再分配（可选）

**说明**：OCP POC 的共识权威来自**资本锁定**，存款即已构成承诺，协议层不要求「必须交易」。本小节为**应用层可选机制**：若希望激励流动性、提高交易深度，可启用；否则可仅按本金占比分配手续费。

| 项目 | 规则 |
|------|------|
| **躺平定义** | 本分红周期内在本预测市场**无任何交易** |
| **躺平者分红** | 应得份额 × 30%（可调参数 $\theta$） |
| **让出部分** | 应得份额 × 70%，按「本周期有交易者」的本金占比再分配 |

### 2.4 三态输出与市场形态

**OCP 协议层三态**：最终共识（Finalized Consensus）仅有三种结果——YES、NO、INVALID（见 OCP §2.1）。应用层可对应三种市场形态：

- **真相明确态**：多数掌握真实信息，套利者资金涌入真相方
- **谎言操纵态**：大户试图扭曲结果，手续费泵将成本转化为金库收入，理财者追逐高 APY 流向真相方
- **混沌迷雾态**：真相扑朔迷离，巨大资本偏差形成「悬赏」，知情者可进场挑战收割

## 3. 理论论证（应用层）

在**典型条件**下（事件可验证、信息可获知、信息分散、资本可流动、参与者逐利），可论证「资本逐利 → 正确预测产生利润 → 资本有动力流向真相」。预测市场支付结构（对则 1、错则 0）使正确预测产生正期望利润；孔多塞陪审团定理、挑战机制信号价值、预测市场文献共同支撑「有信息资本在对决期偏向真相」。详见原白皮书理论部分（可归档为独立理论附录）。

**重要**：协议输出是**资本置信度结果**（资本对某一结果的最大可承受承诺），而非客观真理裁决；与 OCP POC 定义一致。

## 4. 参数与边界

### 4.1 可调参数

| 参数 | 作用 | 调参建议 |
|------|------|----------|
| 挑战保证金 $b$（= 金库最小存款额） | 过滤 frivolous 挑战；由金库发起人设定 | 过低→噪音多；过高→真实挑战被抑制 |
| 冷静期/再投票期时长 $\Delta$ | 挑战窗口与再投票期时长（OCP 由应用层配置） | 1 小时为参考值 |
| 手续费率 $f$ | 金库收入与交易成本平衡 | 0.3% 量级 |
| 躺平分红比例 $\theta$（可选） | 躺平者获得份额系数；未启用躺平惩罚时无效 | 30% |

### 4.2 适用边界

- 高信息透明度事件效果最佳
- 截止日可验证；长期不可验证事件超出设计范围
- 足够流动性；冷启动需激励
- **父子事件**：若事件依赖前提命题（如「比赛是否正常进行」），可引用 OCP 父子事件（父事件 INVALID 则子事件自动退款）；见 [OCP 白皮书](./OCP_WHITEPAPER_CN.md) §2.7
- **事件历史账本**：协议层维护事件历史与参与者历史（赢方质押、挑战记录等），用于激励与可审计性；见 OCP §2.8

### 4.3 无效与结局未定处理

遵循 OCP 协议层三态规则。本应用层存在两类「无效」情形：

| 类型 | 触发条件 | 处理 |
|------|----------|------|
| **命题无效**（协议层 INVALID） | 比赛取消、真相无从考证、命题歧义；经 INVALID 挑战与再投票期后 INVALID 胜出 | 按 OCP §2.6：发起人 b 与挑战者 b 退还，质押者本金原路退回；分红按原金库逻辑提取 |
| **分布未定**（应用层） | 再投票期结束后 YES/NO 权重均未超 50%（含 50:50） | 协议层判 INVALID 胜出，全员退款；见 OCP 三态终局规则 |

## 5. 失败模式与事件排除

### 5.1 失败模式（概要）

- 资本并不总是追随真相（信息不可验证、意识形态主导时）
- 「真相存在但无人挑战」的稳定均衡
- 系统收敛的是资本置信度，非客观真相
- 若启用躺平惩罚，可能提升交易量但不保证信息质量
- 无法区分「不可知」与「错误」的事件

### 5.2 不适合的事件

| 类型 | 示例 | 原因 |
|------|------|------|
| 不可终止验证 | 「某人是否永远诚实」 | 挑战机制无法自然终止 |
| 私密不可证 | 秘密协议、内部决定 | 信息优势无法转化为资本优势 |
| 强意识形态 | 宗教真理、政治立场 | 资本反映身份认同，非事实 |
| 自我实现 | 「该市场是否选 A」 | 结果可被博弈直接操纵 |

### 5.3 适合事件的正向定义

- 有明确截止点
- 可在对决期被公开验证
- 信息可通过市场传播
- 真相方存在潜在套利空间

## 6. 与现有方案对比

| 维度 | Polymarket | Augur | CCP 预测市场 |
|------|------------|-------|--------------|
| 结算依赖 | UMA 预言机 | 报告者 + REP | CCP 协议（无预言机） |
| 挑战机制 | 2hr 异议 | dispute | 一次性挑战 + 保证金 |
| 资本承诺 | 无锁仓 | REP 质押 | 锁定至结算 |
| 争议处理 | 链下委员会 | 多轮 dispute | 一次性对决期后终局 |

---

## 7. 漏洞与缓解

| 漏洞 | 缓解 |
|------|------|
| 最小交易刷资格 | 可设最小交易量门槛 |
| Griefing | 一次性挑战天然限制（无多轮拖延） |
| 闪电贷 | 存款锁定至结算，无法同一 tx 还款 |
| 全躺平（仅当启用躺平惩罚时） | Fallback：再分配池滚入下一周期或返还 |
