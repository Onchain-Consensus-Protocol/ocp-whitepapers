# 资本承诺协议：一种生成昂贵社会共识的机制

## Capital Commitment Protocol: A Scheme for Costly Social Consensus

> **实现状态**：本文档定义协议机制（CCP）。共识上链的叙事框架见 [OCP 白皮书](./OCP_WHITEPAPER_CN.md)。预测市场应用详见 [PREDICTION_MARKET_APPLICATION_SPEC.md](./PREDICTION_MARKET_APPLICATION_SPEC.md)。

---

## 摘要

本文提出一种**三元结构**的链上协议，用于形成关于外部命题的、抗操纵的资本共识。协议通过三阶段实现：

1. **质押**：参与者以资本为信念加权，形成初始分布。
2. **挑战**：系统允许少数派支付成本，对临时多数发起「压力测试」，开启再投票窗口。
3. **共识**：当且仅当一方资本权重能抵御所有有利可图的挑战后，协议输出最终共识。

该协议不生产真相，而是共识的**「压力测试熔炉」**，为预测、仲裁与治理提供无需信任的基础层。其底层逻辑即**承诺证明（POC）**：共识的权威性由资本的可验证锁定来证明。

**关键词**：质押、挑战、共识、资本承诺协议、承诺证明（POC）、赢家通吃、博弈论

## 1. 引言：共识的代价

### 1.1 问题

现有的去中心化系统如何对外部事实或主观选择做出**不可逆的、抗操纵的集体承诺**？

预言机依赖链下权威或复杂治理；链上法庭（如 Kleros）依赖多数投票；预测市场（如 Polymarket、Augur）依赖外部结算来源。共同问题是：**真相或终局裁决依赖外部裁判**，存在单点信任与操纵风险。

### 1.2 核心论点

最高形式的可信承诺，是**愿意为其承担巨大的、可验证的财务风险**。本协议旨在将这一原则代码化：不引入外部裁判，而让**资本本身**在博弈中形成共识；该共识的价值，恰恰来自其经受过高成本挑战的考验。

### 1.3 协议哲学

**本协议不生产真相，只生产最昂贵的共识。** 你想用它来做什么——预测事件、仲裁争议、治理表决、传递信念——是应用层的事。

**本协议是一个将质押转化为共识的博弈机器，而挑战是其唯一的、必要的转换引擎。**

## 2. 机制设计：质押 — 挑战 — 共识

### 2.1 质押阶段

**质押**：个体将其资本在特定时间段内，单向绑定于某一命题之上的行为。这是信念的信号与成本。（*哲学隐喻：投入土壤的种子。是原料，但本身并非果实。*）

- **三态事件**：命题有三个互斥结果 YES、NO、INVALID（事实破裂/前提不存在/无法验证）
- **截止时间 $t_0$**：质押期结束的时刻，此后进入挑战阶段
- **参与者**：任何持有可存入资产的地址，可向选项 YES、NO 或 INVALID 存入资金，**完全锁定**

质押期形成的初始权重分布，即个体信念的资本化表达。领先方 = 资金权重 > 50% 的一方（仅计 YES 与 NO 时，用于判定 Y/N 挑战权）。

### 2.2 挑战阶段

**挑战**：支付额外成本，申请对现有质押分布进行一次公开的、有限时间的重新竞争的权利与过程。（*哲学隐喻：压力测试与自然选择。是淬炼、优胜劣汰的过程。*）

- **Y/N 挑战权**：仅属劣势方（YES/NO 中资本权重少的一方）
- **INVALID 挑战权**：任何人可发起，声称事件无法判定（比赛取消、真相无从考证等）
- **保证金 $b$**：协议定义 b = 金库最小存款额；金库发起人设定存款代币与最小存款额。b 为绝对金额，以存款代币计价，无需预言机
- **挑战窗口**：截止后固定时长 Δ（如 1 小时）
- **开放再投票期**：挑战触发后，新资本可流入 YES、NO 或 INVALID；**原有质押者可重新投票、重新选择立场**。这是协议的**动态安全核心**，将静态质押转化为动态博弈

**一次性终局**：协议层仅此一轮。挑战窗口结束若无挑战 → 立即终局；若有挑战，再投票期结束后 → 立即终局，不再接受任何新挑战。此规则由协议定义，不可由应用修改，确保 Finalized Consensus 的全球一致性与权威性。

### 2.3 共识形成阶段

**共识**：当质押分布能够持续抵御任何有利可图的挑战后，所涌现出的稳定的终局状态。（*哲学隐喻：幸存者的奖杯。是最终产物，代表了当前最高成本的信念网络。*）

- **终止条件**：挑战窗口期结束且无人挑战 **或** 再投票期结束（若有挑战）
- **三态终局**：
  - YES > 50% → YES 胜出，YES 方通吃 NO 方资金（含对手保证金）
  - NO > 50% → NO 胜出，NO 方通吃 YES 方资金（含对手保证金）
  - 否则 → INVALID 胜出，全员退款，分红按原金库逻辑提取
- **INVALID 失败时**：挑战者 b 没收，按比例分给金库存款人（不含挑战者本人）

```
┌─────────────────────────────────────────────────────────────────┐
│  质押阶段（至 t₀）                                               │
│  参与者向 YES、NO 或 INVALID 存入资金，完全锁定；形成信念分布     │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  挑战阶段（挑战窗口 Δ）                                          │
│  Y/N 劣势方或任何人可支付 b 发起挑战；无人挑战 → 窗口结束即共识   │
└─────────────────────────────────────────────────────────────────┘
                              │
                    有人挑战
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  开放再投票期（Δ）：新资本可流入 YES/NO/INVALID；可重新投票       │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  共识形成：YES>50% 或 NO>50% 通吃；否则 INVALID 全员退款；终局    │
└─────────────────────────────────────────────────────────────────┘
```

### 2.4 父子事件与历史

为提高共识对现实世界的准确度，协议支持**父子事件引用**：若父事件（如「比赛进行」）终局为 INVALID，子事件（如「湖人赢」）自动触发退款。协议维护**事件历史账本**与**参与者历史**（赢方质押累积等），激励用户维护真相。

## 3. 理论分析：挑战的转换器作用

本节的中心从「资本是否趋真」，转移到**「挑战机制如何将脆弱的质押组合，锻造为坚固的共识」**。挑战是质押与共识之间的唯一转换引擎。

### 3.1 博弈论模型：挑战的严肃性

成功的挑战必须同时具备**（a）高私人信息置信度** 与 **（b）吸引边际质押的能力**。这过滤了噪音，确保了挑战的严肃性。

设挑战者主观概率 $q = \mathbb{P}(T=\text{己方})$，挑战成功后可得总期望收益 $\pi$，挑战成本 $c$，保证金 $b$，则

$$
\mathbb{E}[U_{\text{challenge}}] = q(\pi - c) - (1-q)(b + c)
$$

挑战发生当且仅当 $\mathbb{E}[U_{\text{challenge}}] \geq 0$，即

$$
q \geq \frac{b+c}{\pi+b}
$$

**结论**：在理性假设下，只有**高置信度**且**预期能吸引再投票**者才会挑战。保证金 $b$ 过滤噪音，使得 frivolous 挑战在经济上不理性。

### 3.2 安全哲学：挑战是共识的「免疫系统」

系统的最终安全性不在于质押的大小，而在于**共识经历挑战后仍能存活**。

- **挑战是共识的免疫系统**：脆弱的质押组合会在挑战中被淘汰；只有能吸引足够再投票、抵御挑战的一方，才能形成终局共识。
- **攻击行为增强系统**：恶意挑战者若失败，保证金奖励给防御方。攻击系统的行为会向防御者输送财富，使系统呈现**反脆弱性**。

### 3.3 再投票期资金流

设再投票期新增权重 $\Delta W_A, \Delta W_B$。在**典型条件**下（信息可获知、资本逐利、有足够有信息者参与），新增质押在统计上偏向真相：

$$
\mathbb{E}[\Delta W_A - \Delta W_B \mid T=A] > 0,\quad \mathbb{E}[\Delta W_A - \Delta W_B \mid T=B] < 0
$$

即便仅有一部分新增资金为有信息资本，只要其信号不被噪音完全淹没，上述偏置在期望意义下仍成立。故在典型条件下，再投票期后的领先方更可能反映真相；协议输出可被视为**资本置信度结果**。

### 3.4 与 Schelling Point 的对比

本协议是一个**资本化、可争议的 Schelling Game**：

| Schelling Game | 资本承诺协议 |
|----------------|--------------|
| 投票权重 | 质押权重 |
| 协调成本 | 锁仓 + 保证金 |
| 协调失败 | 直接资本损失 |
| 协调焦点 | 「谁能赢得再投票竞争」 |

协调失败者承担真实、持续的资本成本。可视为 **Capital-Weighted Schelling Game with One-Shot Challenge**。

## 4. 协议属性与安全模型

### 4.1 无需预言机

共识完全内生。结算结果由链上资本分布决定，不依赖任何外部数据源或裁判。

### 4.2 最小化信任

只信任代码和参与者自身的逐利动机。无中心化管理员，无链下委员会。

### 4.3 安全边界

**明确承认**：当资本被系统性误导（如极端意识形态事件、信息不可公开验证、信号严重相关）时，协议输出的是**「昂贵错误」**而非真相。协议不声称生产真理，只声称生产**经高成本考验的共识**。适用域由应用层的事件选择规则界定（见第 5 节及应用规格文档）。

## 5. 应用场景：质押—挑战—共识的实例化

以下应用均为「质押—挑战—共识」三阶段流程的具体实例化。**本节仅列举，详细机制见各应用规格文档**。

### 5.1 预测市场

完整的「质押（交易与存款）— 挑战（争议）— 共识（结算）」循环：

- **质押**：用户买卖 YES/NO 份额、向金库存款，形成初始信念分布
- **挑战**：输家可支付保证金发起挑战，开启再投票窗口，新资本可流入
- **共识**：再投票期结束，领先方赢得全部，结算

详见 [PREDICTION_MARKET_APPLICATION_SPEC.md](./PREDICTION_MARKET_APPLICATION_SPEC.md)。

### 5.2 去中心化仲裁

- **质押**：原告/被告将抵押品存入 A/B
- **挑战**：败诉方挑战判决，吸引「陪审团」（新资本）在再投票期流入
- **共识**：再投票期结束，形成终局裁决

### 5.3 重大治理投票

- **质押**：参与者对提案投票，资本锁定于赞成/反对
- **挑战**：反对派可通过挑战试图颠覆初始投票质押结果
- **共识**：能抵御挑战的质押分布，确保执行的共识具有极高韧性

### 5.4 信念信号与承诺

纯粹为展示社区对某命题的信心强度。结果本身（钱给谁）甚至不重要，重要的是**质押与挑战过程中显示的承诺硬度**。

## 6. 讨论、局限与未来工作

### 6.1 参数影响

| 参数 | 设定方 | 过小 | 过大 |
|------|--------|------|------|
| $b$（= 金库最小存款额） | 金库发起人 | 噪音挑战增多 | 真实挑战被抑制 |
| $\Delta$（挑战窗口/对决期时长） | 应用层 | 信息聚合不足 | 资本效率下降 |

### 6.2 局限

- **低流动性**：参与者少时，共识的「硬度」有限
- **高度模糊事件**：当信息不可获知或资本被系统性误导时，协议输出可能偏离真相
- **「真相存在但无人挑战」**：若知情者预期对决期新资本仍将站在对方，其最优策略是不挑战，真相可能永远不被揭示

### 6.3 未来工作

- 跨链实现
- 保证金与领先方资金的比例化设计
- 新应用形式（治理、仲裁的详细规格）

## 7. 结论

资本承诺协议（CCP）提供了一种将**社会信念「资本化」并「压力测试」**的基础工具。其输出是社会认知的**硬度的衡量**，而非绝对真理。

**本协议是一个将质押转化为共识的博弈机器，而挑战是其唯一的、必要的转换引擎。**

比特币白皮书定义了「无需信任的价值转移」；本协议则定义「**无需信任的昂贵承诺**」——前者是货币的基石，后者是广义社会协调的基石。
